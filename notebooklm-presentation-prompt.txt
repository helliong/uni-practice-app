Создай на русском языке академическую презентацию для защиты учебного проекта.

Тема:
«Разработка веб-приложения — интернет-магазина по продаже университетской и IT-атрибутики»

Докладчик:
Мирошниченко Егор Юрьевич

Продолжительность выступления: 8 минут.
Количество слайдов: 9–10.
Аудитория: преподаватель и студенты технического направления.

Используй только сведения из загруженных источников проекта. Не придумывай функции, технологии, результаты и числовые показатели, которых нет в документации.

Структура презентации:

1. Титульный слайд
   - тема проекта;
   - ФИО докладчика;
   - тип работы: разработка веб-приложения.

2. Актуальность и проблема
   - университетская и IT-атрибутика представлена в разных источниках;
   - пользователю требуется единый удобный каталог;
   - необходимы поиск, оформление заказов, личный кабинет и административное управление.

3. Цель и задачи проекта
   - разработать полнофункциональный прототип интернет-магазина;
   - реализовать каталог, поиск, корзину и избранное;
   - добавить регистрацию и авторизацию;
   - реализовать оформление и оплату заказов;
   - создать административную панель;
   - обеспечить удобный запуск и хранение данных.

4. Использованные технологии
   - Next.js 16 и React 19;
   - TypeScript;
   - SCSS/Sass;
   - PostgreSQL и Prisma ORM;
   - NextAuth.js и bcryptjs;
   - Zustand и React Context;
   - Nodemailer;
   - AWS SDK и S3-совместимое хранилище;
   - Docker Compose, MinIO и Mailpit.
   Представь технологии компактно, желательно в виде схемы или групп: frontend, backend, данные, инфраструктура.

5. Архитектура приложения
   Покажи понятную схему:
   пользователь → Next.js-интерфейс → API routes → Prisma → PostgreSQL.
   Отдельно покажи взаимодействие приложения с MinIO/S3, Mailpit/SMTP и платёжным сервисом.
   Объясни назначение frontend, серверных маршрутов, базы данных и внешних сервисов.

6. Основные пользовательские возможности
   - каталог университетской и IT-атрибутики;
   - поиск, фильтрация и просмотр карточек товаров;
   - выбор размера и цвета;
   - корзина и избранное;
   - регистрация с подтверждением email;
   - личный кабинет;
   - оформление заказа;
   - подтверждение статуса студента;
   - адаптивный интерфейс и светлая/тёмная тема.
   Не перегружай слайд текстом — используй короткие тезисы и визуальные блоки.

7. Административные и серверные возможности
   - управление товарами и вариантами;
   - управление заказами;
   - обработка заявок на подтверждение статуса студента;
   - журнал действий администраторов;
   - серверная проверка цен, остатков и входных данных;
   - разграничение доступа по ролям.

8. Docker и автономный запуск
   Покажи, что после клонирования репозитория проект запускается одной командой:
   docker compose up -d --build

   Объясни, что без обязательного .env автоматически запускаются:
   - приложение;
   - PostgreSQL;
   - Mailpit для тестовых писем;
   - MinIO для локального S3-хранилища;
   - миграции и демонстрационные данные;
   - демонстрационный режим оплаты.

   Укажи адреса:
   - приложение: http://localhost:3001;
   - письма Mailpit: http://localhost:8025;
   - MinIO: http://localhost:9001.

9. Тестирование и результат
   - TypeScript-проверка;
   - ESLint;
   - production-сборка Next.js;
   - 66 автоматических тестов;
   - проверка реального запуска контейнеров;
   - проверка доступности приложения, Mailpit и MinIO;
   - проверка отправки регистрационного письма.
   Сформулируй главный результат: создан работоспособный адаптивный прототип интернет-магазина, который можно автономно развернуть через Docker.

10. Итоги и дальнейшее развитие
   Итоги:
   - реализованы основные пользовательские и административные сценарии;
   - данные хранятся в PostgreSQL;
   - проект подготовлен к простой демонстрации;
   - архитектура допускает подключение реальных внешних сервисов.

   Возможные направления развития:
   - подключение production-магазина ЮKassa;
   - реализация возвратов и электронных чеков;
   - подключение полноценной службы поддержки;
   - расширение тестового покрытия;
   - настройка CI/CD;
   - улучшение серверного поиска и масштабирования каталога.

Требования к оформлению:
- современный минималистичный дизайн;
- спокойная академическая цветовая палитра;
- визуальная связь с университетской и IT-тематикой;
- крупные заголовки;
- не более 5–6 коротких тезисов на одном слайде;
- избегать длинных абзацев;
- использовать схемы, иконки, карточки интерфейса и фрагменты проекта из источников;
- все подписи и заголовки должны быть на русском языке;
- не использовать вымышленные статистические данные;
- Docker-команду и адреса оформить как код;
- последний слайд завершить фразой «Спасибо за внимание».

Дополнительно подготовь заметки докладчика для каждого слайда. Общая продолжительность речи должна составлять около 8 минут. Распредели время примерно по 40–55 секунд на содержательный слайд, титульному и заключительному слайдам удели меньше времени. Текст выступления должен быть естественным, понятным и подходящим для устной защиты, а не для чтения технической документации.
